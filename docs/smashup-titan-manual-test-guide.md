# SmashUp 泰坦机制手动测试指南

## 测试前准备

1. 启动开发服务器：
```bash
npm run dev
```

2. 打开浏览器访问：`http://localhost:5173`（Vite 默认端口）

3. 创建一个 SmashUp 游戏房间

4. **重要**：选择 POD 派系（带泰坦的派系）：
   - 恐龙 (POD版)、巫师 (POD版)、忍者 (POD版)、海盗 (POD版)
   - 杀手植物 (POD版)、诡术师 (POD版)、外星人 (POD版)
   - 熊骑兵、印斯茅斯、克苏鲁、狼人、弗兰肯斯坦、吸血鬼、巨蚁

5. **如何查看游戏状态**（使用浏览器控制台）：
   - 按 F12 键打开控制台
   - 切换到 Console 标签页
   - **首次使用时**：浏览器会显示安全警告 "Don't paste code into the DevTools Console..."
   - 输入 `allow pasting` 并按回车键（绕过警告）
   - 现在可以输入命令查看游戏状态了

---

## 测试场景 1：泰坦初始化

**目标**：验证泰坦卡正确初始化到 titanZone

**步骤**：
1. 选择包含泰坦的派系（如 Bear Cavalry、Pirates、Wizards 等）
2. 游戏开始后，打开浏览器控制台
3. 输入：`window.__BG_STATE__?.core.players['0'].titanZone`
4. 应该看到一个泰坦卡对象数组

**预期结果**：
- titanZone 包含 1 个泰坦卡
- activeTitan 为 null（泰坦还未出场）

---

## 测试场景 2：泰坦出场（Special 能力）

**目标**：验证泰坦可以通过 Special 能力出场

**步骤**：
1. 找到一张泰坦的 Special 卡（如 "Fort Titanosaurus Special"）
2. 打出这张卡
3. 按照提示选择要消灭的随从和出场基地
4. 确认泰坦出场

**预期结果**：
- 泰坦出现在选择的基地上
- titanZone 变为空
- activeTitan 包含泰坦信息（titanUid, baseIndex, powerTokens）

---

## 测试场景 3：泰坦力量计算

**目标**：验证泰坦力量正确计入基地计分

**步骤**：
1. 让泰坦出场到一个基地
2. 在同一基地打出一些随从
3. 让基地达到 breakpoint 触发计分
4. 观察计分结果

**预期结果**：
- 基地总力量 = 泰坦力量（5 + powerTokens）+ 随从力量
- 计分排名正确反映泰坦的贡献

---

## 测试场景 4：泰坦计分资格

**目标**：验证"只有泰坦没有随从"时失去计分资格

**步骤**：
1. 让泰坦出场到一个基地
2. 不要在该基地打出任何随从
3. 让基地达到 breakpoint 触发计分
4. 观察计分结果

**预期结果**：
- 玩家失去该基地的计分资格
- 即使泰坦力量最高，也不能获得 VP

---

## 测试场景 5：泰坦冲突

**目标**：验证两个泰坦在同一基地时触发冲突

**步骤**：
1. 玩家 A 的泰坦出场到基地 1
2. 玩家 B 的泰坦也出场到基地 1
3. 观察冲突结果

**预期结果**：
- 力量较低的泰坦被移除
- 被移除的泰坦回到 titanZone
- powerTokens 被清除
- 如果力量相等，防守方（先到的）获胜

---

## 测试场景 6：泰坦移动（Talent 能力）

**目标**：验证泰坦可以通过 Talent 能力移动

**步骤**：
1. 泰坦出场后，找到对应的 Talent 卡
2. 打出 Talent 卡
3. 选择目标基地
4. 确认移动

**预期结果**：
- 泰坦移动到新基地
- activeTitan.baseIndex 更新
- 如果新基地有对手泰坦，触发冲突

---

## 测试场景 7：Major Ursa Ongoing 能力

**目标**：验证 Major Ursa 移动后可以移动对手随从

**步骤**：
1. Major Ursa 出场到基地 1
2. 对手在基地 1 有战斗力 ≤3 的随从
3. 使用 Talent 移动 Major Ursa 到基地 2
4. 应该弹出交互：选择要移动的对手随从
5. 选择随从和目标基地

**预期结果**：
- 对手随从被移动到选择的基地
- 可以选择"跳过"不移动

---

## 测试场景 8：基地摧毁时泰坦移除

**目标**：验证基地摧毁时泰坦自动移除

**步骤**：
1. 泰坦出场到一个基地
2. 让该基地被摧毁（计分后替换）
3. 观察泰坦状态

**预期结果**：
- 泰坦回到 titanZone
- activeTitan 变为 null
- powerTokens 被清除

---

## 测试场景 9：目标选择（Into the Time Slip）

**目标**：验证可以选择泰坦作为目标

**步骤**：
1. 泰坦出场到一个基地
2. 打出 "Into the Time Slip" 卡
3. 应该可以选择泰坦作为目标
4. 选择泰坦并确认

**预期结果**：
- 泰坦被移除（回到 titanZone）
- 目标选择 UI 正确显示泰坦

## 调试技巧

### 查看游戏状态
```javascript
// 查看完整状态
window.__BG_STATE__

// 查看玩家 0 的泰坦
window.__BG_STATE__?.core.players['0'].titanZone
window.__BG_STATE__?.core.players['0'].activeTitan

// 查看基地上的泰坦
window.__BG_STATE__?.core.bases[0]
```

### 手动触发命令（调试用）
```javascript
// 注意：这只在本地模式下有效
window.__BG_DISPATCH__({
  type: 'PLACE_TITAN',
  payload: {
    playerId: '0',
    titanUid: 'titan-1',
    baseIndex: 0
  }
})
```
