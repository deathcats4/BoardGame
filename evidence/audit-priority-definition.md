# POD 提交审计优先级定义

## 文档说明

---

## 优先级分类标准

### P0 - 核心逻辑（Critical）

**定义**: 影响游戏核心功能、引擎系统、传输层的关键文件

**审计要求**:
- ✅ 必须逐行审查所有删除
- ✅ 必须验证删除的合理性
- ✅ 必须确认无破坏性变更
- ✅ 必须运行相关测试

**包含范围**:
- 引擎层核心文件（`src/engine/`）
- 传输层文件（`src/engine/transport/`）
- 游戏领域层核心文件（`domain/index.ts`, `game.ts`, `reduce.ts`, `execute.ts`）
- 服务端核心逻辑（`src/server/storage/`）
- 框架层核心组件（`src/components/game/framework/`）

**文件数量**: 约 50 个

---

### P1 - 业务逻辑（High）

**定义**: 影响游戏业务逻辑、能力系统、UI 交互的重要文件

**审计要求**:
- ✅ 必须审查关键删除
- ✅ 必须验证业务逻辑完整性
- ⚠️ 可以跳过纯重构/格式化的删除

**包含范围**:
- 游戏能力文件（`abilities/*.ts`）
- 游戏 UI 组件（`ui/*.tsx`）
- 游戏领域层辅助文件（`domain/utils.ts`, `domain/types.ts`）
- 通用 UI 组件（`src/components/common/`）

**文件数量**: 约 80 个

---

### P2 - 测试与配置（Medium）

**定义**: 测试文件、配置文件、国际化文件

**审计要求**:
- ⚠️ 可以批量审查
- ⚠️ 重点关注测试覆盖率变化
- ⚠️ 重点关注配置项删除

**包含范围**:
- 测试文件（`__tests__/*.test.ts`）
- 国际化文件（`public/locales/`）
- 音频配置（`audio.config.ts`）
- 数据文件（`data/*.ts`, `data/*.json`）

**文件数量**: 约 120 个

---

### P3 - 页面与服务（Low）

**定义**: 页面组件、服务层、工具函数

**审计要求**:
- ⚠️ 可以快速扫描
- ⚠️ 重点关注 API 变更
- ⚠️ 重点关注服务端逻辑变更

**包含范围**:
- 页面组件（`src/pages/`）
- 服务层（`src/services/`）
- Context 层（`src/contexts/`）
- 工具函数（`src/lib/`, `src/hooks/`）

**文件数量**: 约 80 个
