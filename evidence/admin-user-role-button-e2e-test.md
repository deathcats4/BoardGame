# 后台用户详情页管理员按钮 E2E 证据

## 目标

验证后台用户详情页新增的“设为管理员 / 撤销管理员”按钮在 UI 层可正常工作，且确认弹窗明确提示“本次操作会写入审计日志”。

## 测试文件

- `e2e/admin-ugc.e2e.ts`

对应新增用例：

- `用户详情页可提升管理员并展示审计提示`

## 执行命令

```bash
npm run test:e2e:ci -- e2e/admin-ugc.e2e.ts
```

## 截图分析

从截图可以确认：

- 用户详情页已正常渲染，用户名为“测试用户”。
- 顶部用户名右侧出现 `ADMIN` 徽标，说明角色提升后的头部状态已刷新。
- 账户信息卡片中的“角色”字段显示为“管理员”，说明详情区同步刷新成功。
- 页面右上角出现“已设为管理员”的成功 Toast，说明前端按钮点击、确认弹窗提交和成功反馈链路完整。
- 页面没有跳回列表或出现空白态，说明角色更新后的重新拉取详情流程正常。

## 结论

本次新增的后台管理员按钮已经完成 UI 验证：

- 可在用户详情页触发角色提升。
- 确认弹窗包含审计日志提示。
- 提交后页面状态和成功提示都会正确更新。
