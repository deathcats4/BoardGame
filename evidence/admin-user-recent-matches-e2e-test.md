# 后台用户详情页近期对局记录 E2E 证据

## 目标

验证后台用户详情页的“近期对局记录”直接使用详情接口 `/admin/users/:id` 返回的 `recentMatches`，不会再被额外的 `/admin/matches?search=<userId>` 空结果覆盖。

## 测试文件

- `e2e/admin-ugc.e2e.ts`

对应新增用例：

- `用户详情页显示详情接口返回的近期对局记录`

## 复现方式

测试里主动构造以下场景：

- `/admin/users/:id` 返回 1 条 `recentMatches`
- `/admin/matches` 即使被请求，也只返回空数组

旧实现会出现：

- 页面顶部统计正常
- 但“近期对局记录”被空列表覆盖，显示“暂无数据”或 `0` 条

修复后要求：

- 页面显示详情接口返回的那条对局记录
- 用户详情页不再依赖 `/admin/matches` 作为近期对局的数据源

## 执行命令

```bash
npm run test:e2e:ci -- e2e/admin-ugc.e2e.ts
```

说明：

- 本次运行中，新增的两个“用户详情页”用例都通过。
- 同文件里原有的 `UGC 包列表/下架/删除流程` 用例存在偶发 `page.goto('/admin/ugc')` 超时抖动，和本次修复无关。

## 结论

“查看用户时近期对局记录无数据”的问题已修复，当前用户详情页以 `/admin/users/:id` 作为单一真实来源，数据链路与页面展示已对齐。
